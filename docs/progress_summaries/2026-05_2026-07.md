# Development Summary: 2026-05-14 to 2026-07-14

Branch: `feat/multi-turn` (58 commits in range; superset of `main`).

## 1. Settings/multi-architecture refactor (May 25-26) — done

Split `settings.py` so architecture-specific code lives under `src/sparq/architectures/v1/`
(nodes, prompts, `system.py`), each with its own TOML config. Done to prepare for a future
v2 architecture without hardcoding v1 assumptions into the settings layer.

## 2. State/REPL concurrency hardening (Jun 1-10) — done

- `State` migrated `TypedDict` → Pydantic `BaseModel`.
- REPL namespace pickles scoped per-run-ID to fix concurrent-run collisions.
- Fixed several REPL bugs: subprocess crashes from top-level pandas/rich imports,
  wrong-namespace writes, submodule alias restoration, traceback source-line display.
- Upgraded LangChain/LangGraph stack to 1.x.

## 3. Parallel step execution in the executor (Jun 16-Jul 2) — done

Added dependency tracking (`depends_on`/`id`) to plan `Step`s. A LangGraph `Send`-based
fan-out (`WorkerState`, `Send`/`END`) was scaffolded then **abandoned**: dependency-batching
was awkward as a dispatcher-loop, and it lost per-worker LangSmith visibility. Replaced with
`asyncio.gather` over dependency-ready batches, with per-step namespace seed/execute/merge/cleanup.
`ExecutorOutput` renamed to `StepResult`, tracked as a list.

The `Send`-based design is documented as revivable if a resumable, reconnect-safe
step-completion tracker is ever needed.

## 4. Sub-task parallelism within a worker (Jul 6-7) — design only, not implemented

Rejected a `spawn_subtasks(subtasks: list[str])` batching tool and `deepagents`' generic
`task` tool (its subagent tools bind once at construction and race on the shared namespace
pickle). Settled on `async_subagent_tool(task: str)` — one sub-task per call, parallelized
by the worker LLM emitting multiple tool calls in one turn, gathered by LangGraph's `ToolNode`.
Introduces `get_ns_lock` to serialize the merge-back race this reopens.

## 5. Executor/worker tooling (Jul 2-8) — mostly done

Switched the worker agent framework twice: `create_react_agent` → `deepagents.create_deep_agent`
→ `langchain.agents.create_agent`, after a Bedrock "grammar too large" error from combinatorial
tool-schema blowup (16 tools) — trading away deepagents' built-in subagent/todo features.
Added: todo tool, retry logic for step failures, statistical-method prompt guidance, friendly
dataset cache paths (fixed a transposed-hash bug), and required tool-verified results (fixed a
fabricated-results bug).

Open, documented-but-unfixed gaps (`docs/improvements.md` #17-19):
- step retries discard prior failure context
- namespace previews in worker context show type only, not shape/values
- context-bloat tracking is by tool-call count rather than token count

## 6. Multi-turn conversation support (Jul 9-13) — design complete, not started

See `docs/multi_turn_support.md`. Key decisions:
- `messages` (via `add_messages`) as the sole history channel, not accumulating
  `query`/`answer` lists.
- `AsyncPostgresSaver` over `AsyncSqliteSaver`/`InMemorySaver`, for pooled concurrent writes
  from parallel workers.
- Deterministic thread-scoped REPL namespace paths with turn-scoped step keys.
- Router heuristic needs to change from topic-membership to "needs new computation" once a
  whole thread is in-domain.
- Estimated 8-11 days of implementation work.
